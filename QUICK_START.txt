## Quick Setup (5 Minutes)

### Step 1: Extract Files
```bash
unzip sarva-github-setup-complete.zip -d sarva
cd sarva
```

### Step 2: Initialize Git Repository
```bash
git init
git add .
git commit -m "feat: initial project setup

Complete Sarva super app infrastructure with:
- Microservices architecture
- CI/CD pipelines
- Linear integration
- Development environment

SAR-001"
```

### Step 3: Create GitHub Repository

**Option A - Using GitHub CLI (Recommended):**
```bash
gh repo create YOUR_ORG/sarva --public --description "Sarva Super App Platform"
git remote add origin https://github.com/YOUR_ORG/sarva.git
git branch -M main
git push -u origin main
```

**Option B - Using GitHub Web:**
1. Go to https://github.com/new
2. Repository name: `sarva`
3. Description: "Sarva Super App Platform"
4. Public/Private: Choose
5. Click "Create repository"
6. Follow the instructions to push existing repository

### Step 4: Create Develop Branch
```bash
git checkout -b develop
git push -u origin develop
```

### Step 5: Import to Linear
1. Open Linear workspace
2. Go to Settings â†’ Import â†’ CSV
3. Upload `sarva_backlog_linear_import.csv`
4. Map columns and import (creates 144 issues in 8 iterations)

### Step 6: Connect Linear to GitHub
1. Install Linear GitHub App: https://github.com/marketplace/linear-app
2. Select your `sarva` repository
3. In Linear: Settings â†’ Integrations â†’ GitHub
4. Configure auto-linking and sync settings

### Step 7: Configure GitHub Secrets
Go to GitHub Repository â†’ Settings â†’ Secrets â†’ Actions

Add these secrets:
- `LINEAR_API_KEY` - Your Linear API key
- `KUBE_CONFIG_STAGING` - Kubernetes config (for deployment)

### Step 8: Enable Branch Protection
1. Settings â†’ Branches â†’ Add rule
2. Branch name pattern: `main`
3. Enable:
   - Require pull request reviews (2 approvals)
   - Require status checks to pass
   - Require branches to be up to date

### Step 9: Start Development
```bash
# Install dependencies
npm install

# Start infrastructure
docker-compose up -d

# Start development
npm run dev
```

## Architecture Overview

Sarva uses a **microservices architecture** with:

- **Host App**: Main container (React Native/Flutter)
- **API Gateway**: Central routing (Kong/AWS API Gateway)
- **Services**: Independent microservices (Node.js/Python/Go)
- **Databases**: PostgreSQL, MongoDB, Redis
- **Message Queue**: RabbitMQ
- **Storage**: MinIO/S3
- **Orchestration**: Kubernetes
